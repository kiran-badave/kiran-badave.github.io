# CV Versions Guide

Your portfolio now includes **TWO optimized CV versions** designed for different purposes:

## 🚀 Product Company CV (1-Page)

### **File:** `cv-product-company.html`

### **Purpose:**
Optimized for **product companies** (Google, Microsoft, Amazon, Flipkart, etc.) targeting **₹30-45+ LPA** salary range.

### **Key Features:**

✅ **Impact-Driven Content**
- Focuses on "What problem did you solve?"
- Quantified achievements (30% improvement, 40% faster, 80% better)
- Business impact highlighted

✅ **Strategic Positioning**
- Title: "Staff Software Engineer | Full Stack (React + Java, Cloud-Native)"
- Emphasizes scalability, system design, cloud-native
- Keywords optimized for ATS (Applicant Tracking Systems)

✅ **1-Page Format**
- Fits on single A4 page
- Easy to scan in 30 seconds
- Print-optimized layout

✅ **New Sections Added**
- **System Design & Architecture** - Critical for senior roles
- **Key Highlights** - Quick wins summary
- Impact metrics throughout

### **Content Highlights:**

**Summary:**
> Staff Software Engineer with 7+ years of experience building scalable, cloud-native applications using React, TypeScript, and Java Spring Boot...

**Experience (Rewritten):**
- "Architected a scalable React-based monorepo, improving code reusability by ~30%"
- "Improved application accessibility (WCAG) by 80%"
- "Led development of ERP modules, improving performance by ~40%"

**System Design Section:**
- Monorepo architecture
- Microservices experience
- Authentication systems (OAuth2, SSO, RBAC)
- Docker & Kubernetes
- CI/CD pipelines

### **Best For:**
- Product company applications
- Senior/Staff Engineer roles
- High-paying positions (₹30L+)
- ATS-optimized applications
- Quick screening rounds

## 🎯 When to Use Which Version?

### **Use Product Company CV (1-Page) When:**
- ✅ Applying to product companies (FAANG, unicorns)
- ✅ Targeting ₹30-45+ LPA roles
- ✅ Applying for Senior/Staff Engineer positions
- ✅ Submitting through ATS systems
- ✅ Need quick, impactful first impression
- ✅ Recruiter has limited time (30-60 seconds)

### **Use Detailed CV When:**
- ✅ Requested by HR for detailed review
- ✅ Interview preparation (have all details handy)
- ✅ Client presentations
- ✅ Background verification
- ✅ Comprehensive portfolio showcase
- ✅ Service companies or consultancies

## 💡 Pro Tips

### **For Product Company Applications:**

1. **Always use Product Company CV first**
   - It's designed for quick screening
   - Optimized for ATS systems
   - Highlights impact immediately

2. **Customize for each application**
   - Keep the format
   - Adjust metrics if needed
   - Add company-specific keywords

3. **Prepare both versions**
   - Start with Product Company CV
   - Have Detailed CV ready for interviews
   - Switch based on request

### **For Interviews:**

1. **Bring both versions**
   - Product Company CV for quick reference
   - Detailed CV for deep discussions

2. **Know your metrics**
   - Be ready to explain "30% improvement"
   - Have examples for each achievement
   - Quantify everything possible

3. **System Design Ready**
   - Product Company CV highlights this
   - Be prepared to discuss architecture
   - Have diagrams/examples ready

## 🎨 Customization Guide

### **To Update Product Company CV:**

1. **Edit `cv-product-company.html`**
   - Update metrics (30%, 40%, etc.)
   - Add new achievements
   - Modify system design section

2. **Keep it 1-Page**
   - Remove less impactful items
   - Combine similar points
   - Use concise language

3. **Focus on Impact**
   - Always quantify results
   - Use action verbs
   - Highlight business value

### **To Update Detailed CV:**

1. **Edit `data.js`**
   - Changes reflect automatically
   - Update experience, skills, education
   - Add new projects

2. **Regenerate if needed**
   - Refresh page to see changes
   - Print to verify layout
   - Check all sections
